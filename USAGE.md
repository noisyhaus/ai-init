# ai-init Usage

## Install location
- Templates: `~/.codex/project-starter/`
- Command: `~/.local/bin/ai-init`

## Basic usage
Run `ai-init` in an empty or existing project directory to create the reusable per-project free-code/Codex starter structure.

Recommended everyday use in free-code:
1. open the target project directory
2. run `ai-init`
3. paste the suggested follow-up prompt into free-code
4. let the agent tailor the generated docs to that specific project before implementation starts

## Options
- `ai-init` : create core files and directories without overwriting existing files; auto-detect new vs existing project for the next-step prompt
- `ai-init --new-project` : force new-project follow-up guidance
- `ai-init --existing-project` : force existing-project follow-up guidance
- `ai-init --auto` : auto-detect mode from current files and folders (default)
- `ai-init --print-prompt` : print only the recommended follow-up prompt for free-code/Codex CLI
- `ai-init --force` : overwrite starter files
- `ai-init --with-rules` : also create starter rule files under `docs/ai/rules/`
- `ai-init --with-memory-plus` : also create `docs/ai/agent_memory.md` and `docs/ai/user_preferences.md`
- `ai-init --with-local-recall` : also create `.ai-index/` scaffolding and `scripts/ai-index.py` for local SQLite FTS recall (implies `--with-memory-plus`)

## Generated by default
- `AGENTS.md`
- `README.md`
- `docs/ai/project_memory.md`
- `docs/ai/change_playbook.md`
- `docs/ai/current_state.md`
- core directories for handoffs, tasks, adr, specs, and plans

## Optional memory-plus files
- `docs/ai/agent_memory.md`
- `docs/ai/user_preferences.md`

## Optional local-recall files
- `.ai-index/README.md`
- `.ai-index/.gitignore`
- `scripts/ai-index.py`

## Prompt files
- New project start: `~/.codex/project-starter/prompts/start-new-project.txt`
- Existing project start: `~/.codex/project-starter/prompts/start-existing-project.txt`
- Session recovery: `~/.codex/project-starter/prompts/start-session.txt`
- Feature addition: `~/.codex/project-starter/prompts/feature-addition.txt`
- Session close: `~/.codex/project-starter/prompts/close-session.txt`

## Recommended free-code flow
### New project
```bash
ai-init --new-project
```
Then paste the printed prompt into free-code.

### Existing project
```bash
ai-init --existing-project
```
Then paste the printed prompt into free-code.

### Let ai-init decide automatically
```bash
ai-init
```
This auto-detects the mode from the current directory and prints the matching follow-up prompt.

### Print only the prompt
```bash
ai-init --print-prompt
```
Useful if the starter files already exist and you only want the next-step prompt again.

## Recovery prompt output format
- `Current objective`
- `Confirmed from docs`
- `Confirmed from live workspace`
- `Unknown or risky points`
- `Next smallest steps`
- `Verification plan`

## ai-init v2 narrow-scope defaults
- Markdown docs remain the source of truth
- Generated `AGENTS.md` carries concise Core Engineering Invariants: SSoT, SRP, Consistency, Atomicity, Idempotency, No Silent Fallback
- `project_memory.md` carries durable structure, canonical sources, and impact rules
- `change_playbook.md` carries change procedures, invariant checks, and verification checklists
- `agent_memory` and `user_preferences` are optional durable memory layers
- `local recall` is an optional extension, not a default requirement
- If `.ai-index/` exists later, treat it as a recall cache only

## Maintainer verification
```bash
~/.codex/project-starter/tests/ai-init-output-test.sh
```

## Session start order
1. run `ai-init` once per project if the starter docs are missing
2. `docs/ai/project_memory.md`
3. `docs/ai/change_playbook.md` if present
4. `docs/ai/current_state.md`
5. optional durable memory and relevant handoff/spec/plan docs

## Recommended close-session behavior
- Collect git evidence first (`git status --short`, `git diff --stat`, `git diff --name-only`)
- Separate in-scope changes from pre-existing dirt, generated noise, and external-path changes
- After classification, read the actual diff content only for in-scope lane files
- If git is not initialized, say so explicitly and treat lane classification as best-effort only
- Update `docs/ai/current_state.md`
- Write `docs/ai/handoffs/YYYY-MM-DD.md`
- If memory-plus is enabled, flush durable lessons into `docs/ai/agent_memory.md`
- If memory-plus is enabled, update stable user preferences in `docs/ai/user_preferences.md`
- If `project_memory.md` structure/canonical-source/impact rules changed, update it too
- If change procedures or verification checklists evolved, update `docs/ai/change_playbook.md`
- If `.ai-index/` exists, note whether it should be updated before the next session
- Git handling should be evidence-first and approval-based: auto-collect and summarize, but do not auto-commit or auto-push without an explicit request

## Local recall commands
```bash
python scripts/ai-index.py rebuild
python scripts/ai-index.py search "Current objective"
python scripts/ai-index.py update
```


## Session close extra output

The close-session flow may also suggest the following at the end:
- a short recovery prompt to paste into the next chat
- the three core files to read next
- recommended staging scope / draft commit title / risks of committing now
